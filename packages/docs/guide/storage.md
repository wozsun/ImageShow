# 存储：本地 / S3 / WebDAV，多后端并存

后端是「命名实例」而非「类型」——同一类型可注册多个（例如两个不同的 S3 桶）。在后台「设置 → 存储」新建并配置：

- **本地存储**：无需额外配置，图片保存在容器的存储目录。
- **S3 兼容存储**：填写 HTTPS endpoint、region、bucket、access key、secret key、根目录与可选 HTTPS 公开访问域名（public base URL），可点击「设为默认上传位置」。
- **WebDAV 存储**：填写 HTTPS base URL、用户名、密码（HTTP Basic）、根目录与可选 HTTPS 公开访问域名（public base URL）。连接 / 首字节超时默认 15 秒，流读取空闲超时默认 15 秒，任务总超时默认 300 秒，三者均可在后端编辑窗口调整；持续收到数据的大文件不会只因传输超过 15 秒被中断。服务器用标准 WebDAV 动词（PROPFIND/MKCOL/PUT/GET/DELETE/COPY）读写，按需自动创建父目录；PROPFIND 响应用 XML parser 解析，并只接受与配置端点同源且位于当前列举根目录内的资源链接，避免将认证信息发送到服务端响应指定的其他地址。临时性 429/5xx、连接 / 首字节 / 空闲超时或网络失败会有限重试和退避，调用方 Abort、Range 流切片和流取消仍向下传递。后端内复制优先使用原生 COPY，异常时退化为 GET+PUT。GET / HEAD 固定请求 `Accept-Encoding: identity`；若服务端忽略 Range 并返回完整 200，ImageShow 用流式跳过与截取产生 206，不缓冲完整对象。**Depth: infinity 列举**默认关闭；关闭时并发递归 `Depth: 1`，开启后单次列举整棵子树，两种方式都有最大结果数保护。

多个后端可同时存在：每张图片记录自己所在的后端（`storage_slug`），读取、缩略图与删除都通过注册表统一解析该图自身的配置和 driver。在**图片编辑窗口**可把单张图迁移到另一后端，在**批量编辑窗口**可批量迁移。批量迁移响应返回 `migrated`、`unchanged` 与 `failed`；管理端仅在 `failed=0` 时按完全成功关闭窗口，部分或全部失败会保留迁移窗口并显示统计结果。在用的后端无法删除；图片、任何尚未清理的导入会话或 `_uploads` 暂存对象均需先移走或清理。**链接图片同样可迁移**——它没有我们的原图，迁移即把其缩略图搬到目标后端（缩略图仍存于独立的 `link/` 前缀，不与本地图的 `thumbs/` 缩略图混放），随后翻转 `storage_slug`。

所有会改变图片对象位置的操作共用“全局共享维护锁 + 单图独占锁”。进入锁后重新读取
当前 `storage_slug`、对象键和分类，先把原图 / 缩略图复制到目标位置，再用旧位置做
数据库 CAS；只有 CAS 成功才删除旧对象。并发分类、存储迁移、主题重分配或彻底删除
因此会串行收敛，失败补偿也只删除本次创建的目标对象。整后端迁移逐图复用同一协议，
不会持有覆盖整个批次的单一长事务。

迁移用统一的物理命名空间 identity 判断两个 slug 是否指向同一位置：S3 由规范化
endpoint、bucket 和 root path 决定，WebDAV 由规范化后的 base URL + root path 决定，
本地后端由运行时存储目录决定；凭据、region、path-style、公开 URL 和超时参数不参与。
identity 相同时迁移会用目标后端凭据回读并逐字节验证共享对象，再切换数据库归属，
绝不复制或删除对象。identity 不同时，
源原图先与数据库 MD5 核对；目标同名对象已存在时必须逐字节等于源对象，不能只凭
`exists` 跳过。新写入的目标对象也会回读校验，完整性确认后才允许数据库 CAS。

非默认后端可以停用；停用后不再作为新图片的写入目标，但已有图片仍可正常读取和迁移。本地存储仅在存在另一个已启用且已设为默认的后端时允许停用，重新启用后才能再次设为默认。

后台存储后端列表按 `type` 返回唯一有效的配置分支：S3 项只含脱敏后的 `s3`，WebDAV 项只含脱敏后的 `webdav`，本地项不携带远端配置。Secret Key / 密码只返回是否已配置的布尔标记，不返回凭据本身。

编辑窗口中的连接测试直接使用表单当前值，不会先保存配置；对于未重新输入的 Secret Key 或 WebDAV 密码，服务端只在本次测试内回填该后端已保存的凭据。测试会写入、读取并在 `finally` 中删除专用探针对象；测试失败不会把尚未确认的字段写入数据库。

S3 的 endpoint、bucket、root path 与 WebDAV 的 base URL、root path 是物理位置字段。
修改时服务端持有全局独占位置锁，并在锁内重新统计该 slug 的全部图片、全部
`import_session`（不按状态或 `expires_at` 排除）和 `_uploads` 对象；任一仍存在即返回
409。导入创建、prepared 暂存写入、commit、取消和过期暂存清理都持有同一位置锁的
共享模式，因此不能在检查后又把暂存写到旧位置。显示名、启停、凭据、region、
path-style、公开 URL 和超时等非物理字段不取得独占锁；访问参数保存前仍会用候选配置
读取一张现有对象并完成临时对象读写自检，避免错误凭据让已有图库整体失联。

对象位置切换后的延迟 `move.cleanup` 会取得同一单图锁，并按当前物理命名空间、前缀和
对象键重新确认所有权；对象若已被后续迁移重新采用则保留，避免旧补偿任务删除当前图片。

本地上传与链接下载的原始字节统一先进入服务端 `data/tmp`。服务端在本地完成校验、标准化、缩略图与最终 MD5 后，才把 processed image 和 prepared thumbnail 写入目标后端 `_uploads`；代理链接只写 prepared thumbnail。因此无需为存储桶配置浏览器 CORS，远端后端也不会发生“上传 raw 后再下载回来处理”的重复传输。详见[功能与流程](./flows#三种图片导入模式)。
